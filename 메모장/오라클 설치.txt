2. 방화벽 포트
    고급 방화벽 : 세밀하게 세팅 (wf.msc) 윈도우 +r -> 인바운드규칙(오른쪽클릭) - 새규칙 - 포트 - 1521
    기본 방화벽 : 쉽게 세팅 (firewall.cpl)

    Oracle : TCP 1521(DataBase) , TCP 8080(Http Listener)
    MS-SQL : TCP 1433
    My-SQL : TCP 3306
Port for 'Oracle Database Listener': 1521
Port for 'Oracle Services for Microsoft Transaction Server': 2030
Port for 'Oracle HTTP Listener': 8080   <== 톰켓 포트(TCP 8080)와 충돌이 발생

3. GUI 관리툴 설치 - ssms
인스턴스 - 하나의 시스템에 SQL Server를 여러번 설치 할 수 있다. 보안이 완전히 분리된 상태
             - 호스팅 서비스, 운영과 개발, 하나의 서버에서 각각 다른 버전의 SQL 서버를 설치할때
	- 기본 인스턴스(MSSQLSERVER) <-- 서버에 처음 설치되는 SQL Server
	- 명명 인스턴스 - 설치 할때 이름을 부여
혼합모드(인증모드) : Window 인증 + sql server 인증
	- Windows 인증 : 현재 Windows에 로그인 한 계정으로 SQL 서버에 접근 (OS 인증, sam에서 인증)
/sam : c:\\window\\system32\\config(계속)\\sam => 암호와 비밀번호가 들어가있음
	- SQL Server인증 : SQL Server의 계정으로 SQL 서버에 접근 (SQL DB에서 인증)
